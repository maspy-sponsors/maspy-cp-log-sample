# 利用について

このレポジトリは，GitHub Sponsors の月額 $3 以上の有料スポンサー向けに限定公開しているものです．

したがって，レポジトリ全体，または多数の `main.cpp` / `memo.md` / `index` をまとめて，第三者に共有・転載・再配布することは許可していません．

有料スポンサー本人による私的利用の範囲で，ソースコードや解説をコピーすることは構いません．

一方で，このレポジトリの内容を集団で共有したり，教材，商用サービス，記事・講座・書籍，AI 学習データなどに利用したい場合には，事前に相談してください．

連絡手段の例は以下です．

- X (Twitter): @maspy_stars
- Codeforces: maspy
- E-mail: [maspypy@gmail.com](mailto:maspypy@gmail.com)

# レポジトリの内容

maspy の競技プログラミングの提出コードと解法メモのログです．

各問題について，提出コード `main.cpp` と短い解法メモ `memo.md` を保存しています．多くのメモは数行程度です．
典型テクニック・類題検索に有用な問題だと判断したものについてはタグを付けている場合があります．

元々は，私が自分のためにつけている記録ですが，需要もあると思うので限定公開します．今後新たに解く問題だけでなく，過去に解いた問題についても，手元の提出コードや記録を整理しながら順次追加していきます．

しばらくは，月に $200$ 問から $1000$ 問程度を目安に追加していく予定です．ただし，追加ペースは状況により変動します．追加実績については [`index/stats.md`](index/stats.md) を参照してください．

## main.cpp

`main.cpp` は問題を解くときに実装しているファイルです．ただしこのまま提出しても Compile Error になります．

提出できる状態にするには，私の個人ライブラリの使用が前提です．`main.cpp` 内から include しているライブラリについては下記レポジトリを参照してください．

https://github.com/maspypy/library

次の expander を使って，include を再帰的に展開すると，提出用の単一コードにできます．

https://github.com/maspypy/library/blob/main/expander.py

ただし，実装の参考にするという用途ではそのような作業は不要だと思います．

また，実装した時点からライブラリの仕様が変更されていることがあります．そのため，古い `main.cpp` は最新の私の個人ライブラリに対応していない可能性があります．

## memo.md

簡単な解説です．

公式解説以上の丁寧な解説をすることは全く目指していません．また，私の考えた解法に基づいているため，公式解説よりも無駄の多い遠回りな解法であることもあります．

解説の粒度は中上級者向けです．丁寧な証明等を書き残すこともありますが，どちらかというと中上級者向け問題に限られます．

## difficulty

難易度は $1$ から $10$ までの $10$ 段階評価です．

おおよそ $150$ 倍（ $120$ 倍から $180$ 倍）程度すれば，AtCoder 配点に対応する，というのが雑な目安になるのかなと思います．

正直そんなにきちんとしていません．時期によっても多少相場は変動していると思います．また，コンテスト開催者の難易度評価や順位表情報などを反映しない数値となっているので，たまたま私が苦戦した，たまたま私がすぐに解けた，といった事象に大きく左右されます．

## tag

いくつかの問題にはタグ付けを行っています．類題を検索しやすくしたり，特定分野の練習問題を集める目的です．

何のタグも付けていないものもあります．おおよそ $10$ 秒程度考えて思いつかなかったら空欄にしていると思います．

タグの目的としては，私自身の検索用が主目的です．そのため一般的なユーザーのためという点ではそれほど精度が高いものではなく，特に汎用的なタグについては一般的なタグ付けまとめサイトの方が充実しています．

一方で，タグ付けの際には非常に出題例が少ない題材で類題を見つけやすくすることは意識しています．そのため，ある問題について少し変わったタグ付けの視点を見つけたり，複数のサイトで多くの問題を解かないと網羅しにくいような題材で類題を探す場合には有用かもしれません．

表記ゆれや，後からより良いタグ名に変更されることもあります．

# index

`index/` 以下に，問題を探すための一覧を生成しています．

* [`index/stats.md`](index/stats.md): 収録数・月次追加数・difficulty 分布
* [`index/recent.md`](index/recent.md): 最近追加した問題
* [`index/difficulty.md`](index/difficulty.md): difficulty 別の収録数と D7 以上の問題一覧
* [`index/tags.md`](index/tags.md): タグ別の問題一覧
* [`index/includes.md`](index/includes.md): `main.cpp` の direct include ごとの問題一覧
* [`index/sites.md`](index/sites.md): サイト・コンテスト別の一覧
* [`index/contests/`](index/contests/): コンテスト単位の一覧
* [`index/problems.tsv`](index/problems.tsv): 機械処理用の全問題一覧

`index/problems.tsv` には，問題 ID，site，contest，problem，title，difficulty，tags，includes，logged_at，memo_path，source_path，url，summary が含まれています．
grep や表計算ソフトなどで検索・集計する場合にはこれが便利だと思います．

特定のサイトやコンテストに収録されている問題をまとめて確認する場合には，[`index/sites.md`](index/sites.md) または [`index/contests/`](index/contests/) から探すことができます．

# 想定用途

## 参加したコンテスト，解いた問題の解説が見たい

このレポジトリに同じ問題が収録されている場合には，`memo.md` から解法の要点を確認できます．また，今後参加するコンテストで私が解いた問題についても，原則としてメモを追加していく予定です．

ただし，解説の粒度は中上級者向けですし，解法の全体や細部を説明していないことも多いです．
公式解説以上の丁寧な解説をすることは全く目指していませんし，特に初心者帯のユーザーが親切な解説を求めている場合には役に立てないことが多いと思います．

公式解説と比べた利点としては，次のような点があります．

- コンテストによっては，公式解説よりも早く解法メモが追加されることがある（Codeforces の Div. 2 など）
- 公式解説とは異なる視点の解法が書かれていることがある
- 海外コンテストについて，日本語の解説を読める

## 問題の実装を見たい

各問題の `main.cpp` から，実際に問題を解く際に記述した実装を確認できます．

多くの `main.cpp` は，コンテスト中やその問題を初めて解いたときの実装をそのまま保存したものです．原則として，ロジックの無駄を整理したり，読者向けに可読性を高めたりする作業は行っていません．

一方で，一部にはコンテスト後に整理した実装も含まれるため，必ずしもコンテスト中の実装と一致するわけではありません．

これらの実装の多くはコンテストへの提出として公開されているものですが，そうでない実装も含まれます．また公開されている提出であっても，ライブラリ include を展開しない形で，実際に記述する部分のコードだけが見られるため，提出ページから実装を探して読むよりも読みやすい可能性があります．

## 難易度やタグから問題を探したい

初心者向けのありふれたテーマについては，他により質の高いまとめサイトが多数あるため，価値はあまり高くないと考えています．

一方で，高難易度問題を探す場合には `index/difficulty.md`，出題があまりないテーマや中上級者向けの典型を探す場合には，`index/tags.md` が役に立つかもしれません．

## ライブラリから問題を探したい

`index/includes.md` に，`main.cpp` の direct include ごとの問題一覧を生成しています．

例えば，

```cpp
#include "ds/segtree/segtree.hpp"
```

を使った問題や，

```cpp
#include "graph/tree_dp/rerooting_dp.hpp"
```

を使った問題を探すことができます．またこれらのライブラリの実装内容は，

https://github.com/maspypy/library

から確認していただくことができます．

これは，特定ライブラリの実戦使用例を探したい場合や，あまりないパターンの練習問題を探したい場合に役立つ可能性があります．

ただし，ここで扱っているのは `main.cpp` に直接書かれている include だけです．ライブラリ内部からさらに include される間接依存は一覧に含めていません．

また，頻出するライブラリについては `index/includes.md` 上では一部のみ表示し，完全な一覧は `index/problems.tsv` の `includes` 列に残す方針にしています．

## より高度な検索を行いたい

レポジトリを clone して `grep` したり，`index/problems.tsv` を処理したりすることで，より高度な検索を行うこともできます．例えば，

- `ds/segtree/segtree.hpp` を `#include` している `main.cpp` のうち，さらに文字列 `seg.max_right` を含むものを検索する
- 特定のタグがつけられている問題のうち，特定の難易度帯の問題を探す
- 特定のライブラリを `#include` している問題のうち，特定の難易度帯の問題を探す

といったことが可能です．
